# Как добавлять и изменять вопросы

Короткая инструкция: где лежат вопросы, в каком формате, как добавить новые
и убрать слишком лёгкие. Можно давать этот файл другому ИИ как образец.

## 1. Где лежат вопросы

Каждый раздел — отдельный файл в папке `js/`:

| Раздел          | Файл                          |
|-----------------|-------------------------------|
| Интегралы       | `js/integrals-questions.js`   |
| Производные     | `js/derivatives-questions.js` |
| Ряды            | `js/series-questions.js`      |
| Пределы         | `js/limits-questions.js`      |
| Дифф. уравнения | `js/ode-questions.js`         |
| Вероятность     | `js/probability-questions.js` |
| Лин. алгебра    | `js/linalg-questions.js`      |

В каждом файле **три массива** — по уровням сложности. Например в `limits-questions.js`:

```js
export const easyLimitsQuestions   = [ /* лёгкие */ ]
export const mediumLimitsQuestions = [ /* средние */ ]
export const hardLimitsQuestions   = [ /* сложные */ ]
```

Эти массивы подключаются автоматически в `js/questions.js` (менять его **не нужно**),
там они собраны в `QUESTIONS[раздел][сложность]`.

## 2. Формат одного вопроса

Один вопрос — это объект `{ ... }`. Пример:

```js
{
  question: "Найдите $\\lim_{x \\to 2} (3x + 1)$",
  options: ["$5$", "$6$", "$7$", "$8$"],
  correct: 2,
  open: ['7'],
  explanation: "Функция линейна: $3\\cdot 2+1=7$."
},
```

Поля:

- **`question`** (обязательно) — текст вопроса. Формулы пиши в долларах: `$...$`.
  Внутри LaTeX (см. §4).
- **`options`** (обязательно) — массив из **4** вариантов ответа (строки, тоже можно LaTeX).
- **`correct`** (обязательно) — номер правильного варианта, **счёт с нуля**:
  `0` = A (первый), `1` = B, `2` = C, `3` = D. В примере выше правильный — `$7$` (третий) → `correct: 2`.
- **`open`** (необязательно) — массив строк, которые засчитываются в режиме «открытый ответ»
  (когда ответ вводят вручную, без вариантов). Указывай все формы записи:
  `open: ['1/2', '0.5', '0,5']`. Если поля нет — открытый ответ для этого вопроса недоступен.
- **`explanation`** (необязательно, но желательно) — краткое объяснение, показывается после ответа.

> ВАЖНО: между объектами ставь запятую. Последний объект в массиве — без запятой в конце
> (или с запятой, JS это допускает, но лучше единообразно).

## 3. Как добавить новый вопрос

1. Открой нужный файл, например `js/limits-questions.js`.
2. Найди нужный массив (`easy…` / `medium… `/ `hard…`).
3. Добавь новый объект в конец массива (перед закрывающей `]`), не забудь запятую
   у предыдущего объекта.

```js
export const hardLimitsQuestions = [
  { question: "...", options: ["$…$","$…$","$…$","$…$"], correct: 1, explanation: "…" },
  // ← добавляй сюда новый вопрос
  { question: "Найдите $\\lim_{x\\to0}\\frac{\\ln(1+x)}{x}$", options: ["$0$","$1$","$e$","$\\infty$"], correct: 1, open: ['1'], explanation: "Табличный предел равен $1$." },
]
```

## 4. Мини-справка по LaTeX (в `$...$`)

В JS обратный слэш пишется **дважды**: `\\`. То есть `\frac` → `\\frac`.

| Что нужно        | Как писать                    | Выглядит как |
|------------------|-------------------------------|--------------|
| Дробь            | `\\frac{a}{b}`                | a/b          |
| Степень          | `x^2`, `x^{10}`               | x², x¹⁰      |
| Индекс           | `x_1`, `\\Delta_x`            | x₁, Δ_x      |
| Корень           | `\\sqrt{x}`, `\\sqrt[3]{x}`   | √x, ∛x       |
| Предел           | `\\lim_{x \\to 0}`            | lim x→0      |
| Интеграл         | `\\int_0^1 x\\,dx`            | ∫₀¹ x dx     |
| Сумма            | `\\sum_{n=1}^{\\infty}`       | Σ            |
| Производная      | `f'(x)`, `\\frac{dy}{dx}`     | f′(x)        |
| Греческие        | `\\alpha \\pi \\lambda \\Delta` | α π λ Δ    |
| Бесконечность    | `\\infty`                     | ∞            |
| Матрица          | `\\begin{pmatrix}1&2\\\\3&4\\end{pmatrix}` | (матрица) |
| Умножение (точка)| `a \\cdot b`                  | a·b          |
| Стрелка          | `\\to`, `\\Rightarrow`        | →, ⇒         |

## 5. Как убрать слишком лёгкие вопросы

Просто удали ненужный объект целиком (всю строку `{ ... },`). Ничего больше править не надо.

Признаки «слишком лёгкого» вопроса, который стоит убрать или усложнить:
- ответ получается в одно арифметическое действие (например `-4 / -2 = 2` без всякой теории);
- в среднем/сложном уровне стоит задача уровня «подставь число».

Лучше не удалять, а **заменить** на содержательную задачу того же уровня — тогда общее
количество вопросов не падает.

## 6. После изменений

- Проверь, что не сломал синтаксис: в терминале `node --check js/limits-questions.js`
  (подставь свой файл). Если ошибок нет — всё ок.
- Обнови страницу приложения — новые вопросы подхватятся автоматически.
- Для быстрой проверки вопроса открой раздел → «Изучение» (study-режим), там видно
  формулировку, варианты и объяснение с формулами.

## 7. Частые ошибки

- Забыл `correct` считать с нуля → правильным помечается не тот вариант.
- Одинарный слэш `\frac` вместо `\\frac` → формула не отрисуется.
- Не 4 варианта в `options` → верстка «поедет».
- Пропущена запятая между объектами → «белый экран» (синтаксическая ошибка);
  лови её через `node --check`.

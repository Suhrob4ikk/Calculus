# question-bank — серверный банк вопросов (источник для submit-test)

Edge Function `submit-test` считает результаты **ежедневки, дуэлей и обычных
тестов** по вопросам из Supabase Storage бакета `question-bank`
(`{subject}_{difficulty}.json`). Клиент рендерит вопросы из `js/*-questions.js`.

**Оба источника обязаны совпадать.** Если они расходятся — серверный подсчёт
ломается. Для ежедневки это особенно критично: она перемешивает easy+medium со
всех разделов по seed от даты; при разной длине массивов один seed даёт **разные
10 вопросов** у клиента и сервера → сервер не находит ответы → **0/10**.

Файлы в этой папке собраны **ровно из текущих клиентских банков** (21 файл,
1644 вопроса) и должны заменить содержимое бакета.

## Как загрузить (один раз, и после каждого изменения вопросов)

**Вариант A — скрипт (проще всего):**
```powershell
$env:SUPABASE_SERVICE_KEY = "<service_role key>"   # Dashboard → Settings → API
node supabase/question-bank/upload.mjs
```
service_role ключ секретный — в код/гит не коммить.

**Вариант B — вручную:** Supabase Dashboard → Storage → `question-bank` →
перетащить все 21 `*.json` из этой папки (с заменой существующих).

## После загрузки
Ежедневка/дуэли/тесты снова считаются корректно. Проверка: пройти ежедневку —
в лидерборде и карточке должен быть верный счёт (не 0/10).

> При изменении `js/*-questions.js` в будущем — регенерировать файлы (команда в
> шапке `upload.mjs`) и снова загрузить.
